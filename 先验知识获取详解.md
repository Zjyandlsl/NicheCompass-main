# NicheCompass 先验知识获取详解

## 📚 概述

NicheCompass 从**三个公开的生物学数据库**获取配体-受体、代谢物-受体等先验知识：

1. **OmniPath** - 配体-受体相互作用
2. **NicheNet** - 配体-受体-靶基因调控
3. **MEBOCOST** - 酶-传感器（代谢物）相互作用

---

## 🔬 数据库 1: OmniPath (配体-受体)

### 数据来源

```python
# 从 OmniPath 数据库获取配体-受体相互作用
import omnipath as op

intercell_df = op.interactions.import_intercell_network(
    include=["omnipath", "pathwayextra", "ligrecextra"]
)
```

### OmniPath 是什么？

**OmniPath** 是一个整合的分子生物学先验知识数据库，包含：
- **数据来源**: 整合了多个公开数据库的细胞间通信数据
- **覆盖范围**: 724 个人类配体-受体相互作用
- **质量控制**: `curation_effort` 参数（至少被2篇论文描述）

### 获取流程

```
┌─────────────────────────────────────────────────────────┐
│  1. 从 OmniPath 在线数据库下载                          │
│     op.interactions.import_intercell_network()          │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  2. 过滤只保留配体-受体相互作用                         │
│     category_intercell_source == "ligand"              │
│     category_intercell_target == "receptor"            │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  3. 按配体分组，收集所有对应受体                         │
│     例如: VEGFA → [FLT1, KDR, FLT4]                      │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  4. 如果是小鼠，映射到小鼠同源基因                       │
│     使用 human_mouse_gene_orthologs.csv                │
│     (从 Ensembl BioMart 获取)                           │
└─────────────────────────────────────────────────────────┘
```

### 示例数据

```
配体 → 受体:
VEGFA → FLT1, KDR, FLT4 (血管生成)
WNT3A → FZD1, FZD2, FZD3, FZD4, FZD5, FZD6, FZD7, FZD8, FZD9, FZD10 (WNT 信号)
TGFB1 → TGFBR1, TGFBR2, ACVR1, ACVR2A (TGF-β 信号)
```

## 🧬 数据库 2: NicheNet (配体-受体-靶基因)

### 数据来源

```python
# 从 NicheNet 下载两个文件
lr_network_url = "https://zenodo.org/record/7074291/files/lr_network_mouse_21122021.rds"
ligand_target_matrix_url = "https://zenodo.org/record/7074291/files/ligand_target_matrix_nsga2r_final_mouse.rds"
```

### NicheNet 是什么？

**NicheNet** 是一个包含：
- **配体-受体网络**: 已知的配体和它们的受体
- **配体-靶基因矩阵**: 每个配体对下游靶基因的调控潜力分数

### 获取流程

```
┌─────────────────────────────────────────────────────────┐
│  1. 从 Zenodo 下载 NicheNet 数据                         │
│     - lr_network.rds (配体-受体网络)                     │
│     - ligand_target_matrix.rds (调控潜力矩阵)            │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  2. 过滤靶基因                                          │
│     根据调控潜力分数筛选:                               │
│     - keep_target_genes_ratio (全局阈值)                │
│     - max_n_target_genes_per_gp (每个 GP 最多250个)      │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  3. 构建基因程序                                        │
│     每个 GP 包含:                                       │
│     - sources: [配体]                                   │
│     - targets: [受体] + [高分靶基因]                    │
└─────────────────────────────────────────────────────────┘
```

### 示例数据

```
配体 → 受体 + 靶基因:
WNT3A → [FZD1, FZD2, ...] + [MYC, CCND1, AXIN2, ...]
       (受体)          (下游靶基因)

VEGFA → [FLT1, KDR, FLT4] + [NOS3, KDR, FLT1, ...]
       (受体)            (下游靶基因)
```

## 🧪 数据库 3: MEBOCOST (酶-传感器代谢物)

### 数据来源

```python
# 从本地文件读取（数据已包含在 NicheCompass 包中）
metabolite_enzymes_df = pd.read_csv(
    "mouse_metabolite_enzymes.tsv", sep="\t"
)
metabolite_sensors_df = pd.read_csv(
    "mouse_metabolite_sensors.tsv", sep="\t"
)
```

### MEBOCOST 是什么？

**MEBOCOST** 基于 Human Metabolome Database (HMDB)，包含：
- **代谢物-酶关系**: 哪些酶产生哪些代谢物
- **代谢物-传感器关系**: 哪些基因检测哪些代谢物
- **物种**: 小鼠 (115 个相互作用) 和人类 (116 个相互作用)

### 获取流程

```
┌─────────────────────────────────────────────────────────┐
│  1. 读取代谢物-酶文件                                   │
│     只保留代谢物作为"产物"的反应                         │
│     direction == "product"                               │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  2. 读取代谢物-传感器文件                               │
│     检测该代谢物的传感器基因                             │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  3. 按代谢物名称匹配                                    │
│     酶基因 + 传感器基因 → 构建基因程序                  │
└─────────────────────────────────────────────────────────┘
```

### 示例数据

```
代谢物: 葡萄糖 (Glucose)
  酶基因: [HK1, HK2, GCK, PFKM, ...]  (糖酵解酶)
  传感器基因: [SLC2A1, SLC2A2, SLC2A3, ...]  (葡萄糖转运体)

代谢物: 乳酸 (Lactate)
  酶基因: [LDHA, LDHB, ...]
  传感器基因: [SLC16A1, SLC16A3, ...]

代谢物: 谷氨酸 (Glutamate)
  酶基因: [GLS, GLS2, ...]
  传感器基因: [GRM1, GRM2, GRM3, ...]  (谷氨酸受体)
```

### 合并规则

```
原始 GP 数量:
  OmniPath: ~2500
  NicheNet: ~1500
  MEBOCOST: ~115
  总计: ~4115

合并规则:
  1. 相同来源基因的 GP 合并
  2. 目标基因重叠度 100% 的 GP 合并
  3. 去除冗余 GP

最终 GP 数量: ~297
```

### 合并示例

```
合并前:
  VEGFA_ligand_receptor_GP
    sources: [VEGFA]
    targets: [FLT1, KDR, FLT4]

  VEGFA_ligand_receptor_target_gene_GP (来自 NicheNet)
    sources: [VEGFA]
    targets: [FLT1, KDR, FLT4, NOS3, KDR, ...]

合并后:
  VEGFA_combined_GP
    sources: [VEGFA]
    targets: [FLT1, KDR, FLT4, NOS3, ...]  # 合并所有目标基因
```

## 💡 关键点总结

### 1. 数据来源的多样性

| 数据库 | 类型 | 覆盖范围 | 用途 |
|--------|------|----------|------|
| **OmniPath** | 配体-受体 | 724 个相互作用 | 经典的细胞间通信 |
| **NicheNet** | 配体-受体-靶基因 | ~1500 个 GP | 完整信号通路 |
| **MEBOCOST** | 酶-传感器 | ~115 个相互作用 | 代谢相关通信 |

### 2. 数据质量控制

- **OmniPath**: `curation_effort >= 2` (至少2篇论文支持)
- **NicheNet**: 调控潜力分数阈值 + 每个GP最多250个靶基因
- **MEBOCOST**: 只使用 HMDB 数据库的代谢物

### 3. 物种处理

- 原始数据主要是人类
- 使用 Ensembl BioMart 进行人-小鼠基因映射
- 一个人类基因可能对应多个小鼠同源基因

### 4. 合并策略

- 相同来源基因的 GP 合并
- 目标基因重叠度高的 GP 合并
- 去除冗余和子集 GP

## 🚀 如何更新数据？

### 方式 1: 从数据库重新下载

```python
# OmniPath - 总是获取最新数据
omnipath_gp_dict = extract_gp_dict_from_omnipath_lr_interactions(
    species="mouse",
    load_from_disk=False,  # 从在线数据库获取
    save_to_disk=True      # 保存到本地
)

# NicheNet - 使用指定版本
nichenet_gp_dict = extract_gp_dict_from_nichenet_lrt_interactions(
    species="mouse",
    version="v2",  # 或 "v1"
    load_from_disk=False,
    save_to_disk=True
)
```

### 方式 2: 使用本地缓存

```python
# 使用已下载的本地文件（更快）
omnipath_gp_dict = extract_gp_dict_from_omnipath_lr_interactions(
    species="mouse",
    load_from_disk=True,  # 从本地加载
    lr_network_file_path="path/to/omnipath_lr_network.csv"
)
```

## 🎯 实用建议

1. **首次使用**: 让程序自动下载数据（需要网络连接）
2. **后续使用**: 使用本地缓存（更快，不需要网络）
3. **自定义 GP**: 可以基于这些数据库创建自己的 GP 字典
4. **数据更新**: 定期从数据库更新以获取最新知识
